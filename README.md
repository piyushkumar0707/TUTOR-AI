## Overview

TutorAI is a production-grade, AI-powered tutoring platform built as an intern assignment for an AI startup. The platform integrates a Groq-powered LLM with Retrieval-Augmented Generation (RAG) to deliver accurate, context-aware responses across four core modules: intelligent chat, quiz generation, PDF assignment creation, and a knowledge base retrieval engine.

The UI was designed using **Google Stitch** and converted to React, featuring a dark "Midnight Scholar" aesthetic with animated SVG elements, Material Symbols icons, and responsive layouts.

---

## Features

| Feature | Description |
|---|---|
| **AI Chat** | Ask questions on any topic — RAG context injected into every Groq API call |
| **Quiz Generator** | 5 MCQs dynamically generated by AI on any topic, with score tracking |
| **PDF Generator** | AI-written assignments exported as formatted, downloadable PDFs |
| **RAG Engine** | MongoDB-backed knowledge store with full-text search retrieval |
| **JWT Auth** | Secure register/login with bcrypt password hashing and JWT tokens |
| **Chat History** | Every session saved to MongoDB, viewable from the sidebar |
| **3D Landing Page** | Interactive Three.js orb on the onboarding page (bonus feature) |
| **Animated SVG UI** | Brain and book illustrations on auth pages, Framer Motion transitions |

## Getting Started

### Prerequisites

Make sure you have these installed:

```bash
node --version    # v18+ recommended
npm --version     # v9+
```

You also need:
- A **MongoDB Atlas** account → [mongodb.com/cloud/atlas](https://www.mongodb.com/cloud/atlas) (free tier works)
- A **Groq API key** → [console.groq.com](https://console.groq.com) (free tier, no credit card)

---

### Installation

**1. Clone the repo**

```bash
git clone https://github.com/your-username/tutorai.git
cd tutorai
```

**2. Install backend dependencies**

```bash
cd server
npm install
```

**3. Install frontend dependencies**

```bash
cd ..
npm install
```

---

### Environment Variables

Create a `.env` file inside the `server/` folder:

```env
# MongoDB
MONGO_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/ai_tutor

# JWT
JWT_SECRET=your_super_secret_jwt_key_here

# Groq
GROQ_API_KEY=gsk_your_groq_api_key_here

# Server
PORT=5000
```

> **Where to get these:**
> - `MONGO_URI` → MongoDB Atlas → Connect → Drivers → copy the connection string
> - `GROQ_API_KEY` → [console.groq.com](https://console.groq.com) → API Keys → Create new key
> - `JWT_SECRET` → any random long string (e.g. run `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`)

Create a `.env` file at repo root (optional, for explicit frontend API URL):

```env
VITE_API_URL=http://localhost:5000
```

---

### Seed the RAG Knowledge Base

Before running, seed the 10+ knowledge documents into MongoDB:

Do not run the seeder in production. It is intended for local/dev data setup only.

```bash
cd server
node rag/seeder.js
```

Expected output:
```
Connected to MongoDB
Cleared existing documents
Seeded 3 docs from math.json
Seeded 3 docs from science.json
Seeded 2 docs from history.json
Seeded 2 docs from coding.json
Seeded 2 docs from english.json
Seeding complete!
```

---

### Running the App

**Start the backend server:**

```bash
cd server
npm run dev       # uses nodemon for hot reload
# or
node index.js
```

Server runs at: `http://localhost:5000`

**Start the frontend (in a new terminal):**

```bash
npm run dev
```

Frontend runs at: `http://localhost:5173`

**Vite proxy config** — add this to `vite.config.js` so API calls work:

```js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:5000',
        changeOrigin: true,
      },
    },
  },
});
```

## Module Breakdown

### 1. Auth & Onboarding

| Route | Page | Description |
|---|---|---|
| `/` | `Onboarding.jsx` | Animated landing page with 3D orb |
| `/signup` | `Signup.jsx` | Register with name, email, password |
| `/signin` | `Signin.jsx` | Login — returns JWT stored in localStorage |

- Passwords are hashed with `bcryptjs` (salt rounds: 10)
- JWT tokens expire after 7 days
- `AuthContext` persists login state across page refreshes
- `PrivateRoute` redirects unauthenticated users to `/signin`

---

### 2. AI Chat

| Endpoint | Method | Auth | Description |
|---|---|---|---|
| `/api/chat` | POST | ✅ | Send message → get AI response |
| `/api/chat/history` | GET | ✅ | List all user sessions |
| `/api/chat/history/:id` | GET | ✅ | Get messages in a session |

- Uses **Groq `llama3-8b-8192`** for fast responses
- RAG context injected before every API call
- Topic selector changes the AI system prompt
- All messages saved to MongoDB per session
- Typing indicator shown while waiting for response

---

### 3. RAG Engine

The retrieval engine uses **MongoDB full-text search** with a keyword-fallback:

```
User query → MongoDB $text search → Top 3 relevant chunks → Injected into Groq prompt
```

Knowledge base topics:
- Mathematics (Algebra, Geometry, Calculus)
- Science (Photosynthesis, Newton's Laws, Periodic Table)
- History (World War II, French Revolution)
- Coding (JavaScript, Data Structures)
- English (Essay Writing, Grammar)

To add new documents, create a JSON file in `server/rag/documents/` with this format:

```json
[
  {
    "topic": "Your Topic",
    "title": "Document Title",
    "tags": ["keyword1", "keyword2"],
    "content": "Full text content of the document..."
  }
]
```

Then re-run `node rag/seeder.js`.

---

### 4. Quiz Module

| Endpoint | Method | Auth | Description |
|---|---|---|---|
| `/api/quiz/generate` | POST | ✅ | Generate 5 MCQs on a topic |
| `/api/quiz/save` | POST | ✅ | Save completed quiz result |
| `/api/quiz/history` | GET | ✅ | Get user's past quiz results |

**Flow:**
1. User enters a topic → `POST /api/quiz/generate`
2. Groq returns structured JSON array of 5 questions
3. Questions shown one at a time with A/B/C/D options
4. Correct/wrong feedback shown after each selection
5. Score screen with full breakdown after question 5
6. Result saved to MongoDB

**Groq model used:** `llama3-8b-8192`

---

### 5. PDF Generator

| Endpoint | Method | Auth | Description |
|---|---|---|---|
| `/api/pdf/generate` | POST | ✅ | Generate and stream PDF file |
| `/api/pdf/history` | GET | ✅ | Get user's generated assignments |

**Flow:**
1. User enters topic, grade level, assignment type
2. Groq generates structured assignment content (title, objectives, questions, answer key)
3. `pdfkit` builds a formatted PDF with header, sections, and footer
4. PDF streamed as binary response to frontend
5. Frontend creates a blob and triggers browser file download

**Groq model used:** `llama3-70b-8192` (better quality for long-form content)

---

### 6. 3D Bonus

The landing page features an interactive 3D mesh using **React Three Fiber**:

```bash
npm install three @react-three/fiber @react-three/drei
```

- `IcosahedronGeometry` with `MeshDistortMaterial` (purple glow)
- Wireframe overlay for depth
- `Float` from drei for gentle floating animation
- `OrbitControls` for drag-to-rotate interaction
- `Stars` background with 3000 particles
- Lazy-loaded with `React.lazy` + `Suspense` for performance

## API Reference

### Auth Routes

```http
POST /api/auth/register
Content-Type: application/json

{ "name": "Alex", "email": "alex@example.com", "password": "secret123" }
```

```http
POST /api/auth/login
Content-Type: application/json

{ "email": "alex@example.com", "password": "secret123" }
```

Both return:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": { "id": "...", "name": "Alex", "email": "alex@example.com" }
}
```

### Chat Route

```http
POST /api/chat
Authorization: Bearer <token>
Content-Type: application/json

{ "message": "Explain photosynthesis", "topic": "Science", "sessionId": null }
```

Returns:
```json
{ "response": "Photosynthesis is...", "sessionId": "64abc123..." }
```

### Quiz Route

```http
POST /api/quiz/generate
Authorization: Bearer <token>
Content-Type: application/json

{ "topic": "World War II" }
```

Returns:
```json
{
  "topic": "World War II",
  "questions": [
    {
      "question": "Which year did World War II begin?",
      "options": ["A) 1935", "B) 1937", "C) 1939", "D) 1941"],
      "answer": "C) 1939"
    }
  ]
}
```

### PDF Route

```http
POST /api/pdf/generate
Authorization: Bearer <token>
Content-Type: application/json

{ "topic": "Photosynthesis", "gradeLevel": "Secondary (Grades 6–10)", "assignmentType": "Worksheet" }
```

Returns: Binary PDF file stream (`application/pdf`)

## Deployment

### Deploy Backend to Render

1. Push `server/` to GitHub
2. Go to [render.com](https://render.com) → New Web Service
3. Connect repo, set root to `server/`
4. Build command: `npm install`
5. Start command: `node index.js`
6. Add environment variables from your `.env`

### Deploy Frontend to Vercel

1. Push repo root frontend to GitHub
2. Go to [vercel.com](https://vercel.com) → New Project
3. Connect repo, set root to `/` (repo root)
4. Add environment variable: `VITE_API_URL=https://your-render-url.onrender.com`
5. Update `vite.config.js` proxy target to the Render URL for production
